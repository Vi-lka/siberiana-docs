# Where

Параметр `where` используется для фильтрации. Вам на выбор предоставляются все поля сущности, и в зависимости от типа поля
(`ID`, `String`, `Int`, `Float`, `Boolean`, `Time` и т.д.) и типа Input разные возможности для фильтрации.

## Типы данных
    - `ID` - Уникальный идентификатор в системе
    - `String` - Строковое значение. Записывается в `""`. Пример: `"Hello world"`
    - `Int` - Целочисленный тип данных. Пример: `993`
    - `Float` - Число с плавающей точкой. Пример: `993.7`
    - `Boolean` - Тип данных, принимающий два возможных значения: `true` или `false`
    - `Time` - Дата представленная в формате `Timestamp` `(yyyy-MM-ddTHH:mm:ss.SSSSSSZZZZ)`.  
    Пример: `"2024-09-09T09:59:03.376544Z"` - 9 сентября 2024 г., 16:59:03
    - Другие - Позволяют провалиться в связанные сущности и фильтровать по их полям

## Варианты фильтров

    - Только название поля - строго равняется
    - `Contains` - содержит **с** учетом регистра
    - `ContainsFold` - содержит **без** учета регистра
    - `EqualFold` - равняется **без** учета регистра
    - `NEQ` - не равняется
    - `GT` - больше
    - `GTE` - больше или равно
    - `LT` - меньше
    - `LTE` - меньше или равно
    - `In` - содержится в приведенном массиве
    - `NotIn` - **не** содержится в приведенном массиве
    - `IsNil` - **пустое** значение
    - `NotNil` - **непустое** значение
    - `HasPrefix` - ищет совпадение в **первых** символах строки (с учетом регистра)
    - `HasSuffix` - ищет совпадение в **последних** символах строки (с учетом регистра)
    - `has...` - имеет хотя бы одну **связь** с сущностью
    - `has...With` - выбрать **поле из связанной** сущности
    - `and` - применяет логический оператор `И` ко всем условиям внутри
    - `or` - применяет логический оператор `ИЛИ` ко всем условиям внутри
    - `not` - применяет логический оператор `НЕ` ко всем условиям внутри
